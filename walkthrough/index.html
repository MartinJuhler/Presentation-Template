🚀 GET STARTED
STEP 1

Use the template & open in VS Code

Click "Use this template" on GitHub to create your own repo, then clone and open in VS Code.

# Create your repo from the template on GitHub, then:
$ git clone https://github.com/YOUR-NAME/my-presentations.git
$ cd my-presentations
$ code .
📁 my-presentations/
  ├── 📁 akva-nautilus/ ← example to copy
  ├── 📁 shared/ ← design system
  ├── 📁 .agent/ ← AI skill file
  └── index.html

💡 Copilot automatically reads the .agent/ skill file — no setup needed.

🤖 AI MAGIC
STEP 2

Paste your knowledge & prompt

Open Copilot Chat, paste your content (meeting notes, bullet points, specs, URLs), then ask it to build your slides.

Copilot Chat
YOU @workspace Read .agent/skills/SKILL.md for the design system. Here are my notes: [paste your content]

Create three presentations about [your topic]: a one-pager, a 3-slide pitch, and a full 6-slide deck.
AI I'll create all three formats using the dark design system. Creating onepager.html, presentation-short.html, and presentation.html...

💡 Just paste raw notes, URLs, or key points. The AI does the research and structuring.

👀 INSTANT PREVIEW
STEP 3

Preview, edit & save

Open index.html in your browser to see all presentations. Click ⚡ Edit on any slide to make changes, then 💾 Save to push to GitHub.

Supercharge with MCP & CLI

Give Copilot direct access to your tools. Ask it to configure MCP servers for GitHub, Git, and more — so it can create branches, commit, push, and manage repos from chat.

Copilot Chat — Agent Mode
YOU Set up MCP access for this project:
— GitHub MCP server for repo operations
— Git CLI for version control
— Configure .vscode/mcp.json if needed
AI I'll configure MCP for you. Creating .vscode/mcp.json with GitHub server access. I can now create branches, commit, push, and deploy directly…

💡 With MCP, Copilot can directly create repos, push commits, open PRs, and enable GitHub Pages — all from chat.

⚡
⚠️ OPTIONAL
ADD-ON

Share publicly with GitHub Pages

Enable GitHub Pages and your presentations get a live URL anyone can access. No hosting needed — GitHub does it for you.

⚠️ Warning: GitHub Pages makes your repository content publicly visible. Only enable this if you're comfortable with anyone on the internet seeing your presentations. For private sharing, just open index.html locally — it works perfectly without Pages.
# Ask Copilot to do it for you:
"Enable GitHub Pages on my repo, deploy from main branch"

# Or do it manually:
→ Repo Settings → Pages → Branch: main → / (root)

# Your presentations are now live at:
